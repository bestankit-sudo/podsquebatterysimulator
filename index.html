Assumptions & Data Sources

Datasheets

  • nRF5340 PS v2.0 — MCU + BLE, deep-sleep ~5 uA, BLE TX 3.4 mA
  • nRF7002 PS v1.2 — Wi-Fi 6 companion, ~60 mA active, 15 uA shutdown
  • nPM1100 PS v1.4 — PMIC, 700 nA quiescent (negligible)
  • VL53L4CD DS rev.8 — ToF sensor, 23 mA active, 55 uA ULP per sensor

Efficiency Factors

  • AAA efficiency: 70% — pulse-load ~78% × buck-boost regulator ~87% × temp/aging/cutoff ~94%
  • Li-ion efficiency: 80% — regulator ~90% × aging/cycle ~92% × temperature ~95% × cutoff reserve ~94%
  • Usable mAh = (nominal Wh × efficiency × 1000) / 3.3 V rail

Board-Level Estimates

  • Board deep-sleep: 30 uA practical — nRF5340 ~5 uA + nRF7002 shutdown ~15 uA + PCB leakage
  • BLE TX to e-ink: 4 mA avg — includes RX windows + BLE stack overhead
  • BLE advertising: ~10 uA avg @ 1s interval — always on for e-ink + phone connectivity
  • Li-ion protection IC: ~6 uA — overcurrent/undervoltage protection leakage
  • Sensor on-time per cycle: 1.0 s — 4 sensors measured sequentially
  • Battery self-discharge: ~0.3%/mo Li-ion, ~0.2%/mo AAA — not in formula, adds ~2-5% over product life
  • Nordic DevZone #112208 — community power measurement reference

Design Targets

  • 9-month threshold = acceptable charge/swap cycle for Podsque v1
  • BLE session duration: 3.0 s per e-ink update
  • Wi-Fi session default: 8 s (boot + scan + TX + shutdown)

Assumptions & Data Sources

Display

  • 800x480 monochrome e-paper, 3.9" diagonal
  • Full-screen refresh ~3.5 s at ~100 mA peak (Akash, Hardware Sync)
  • MCU idle-wait during refresh: 1.9 mA (BUSY pin interrupt, on top of 100 mA)
  • Gate drive boost converter (+15V/-15V) included in 100 mA figure
  • E-paper controller standby: ~0.5 uA
  • Zero power to hold image (electrophoretic)
  • Scene JSON primary payload (per BLE payload decision)

nRF52832 (BLE SoC)

  • nRF52832 PS v1.8 — BLE-only SoC for e-ink module
  • BLE TX @ 0 dBm: 5.3 mA (DC/DC), RX: 5.4 mA
  • CPU active @ 64 MHz: ~4.5 mA
  • System ON idle (RAM retention): 1.9 uA
  • Board sleep estimate: ~5 uA (MCU 1.9 + regulator + leakage)

Coin Cell Limitation (Hardware Sync)

  • 100 mA e-paper peak sags coin cell to 2.4-2.5 V — MCU brownout
  • Coin cells: only ~30-45% capacity usable due to voltage dip cutoff
  • Shelf label industry uses 3-4 coin cells in parallel to solve this
  • Supercap workaround degrades over time — unreliable
  • 2x AAA has no peak current limitation — recommended by Akash
  • Cold temperature (0°C): coin cell capacity drops additional 20-30% beyond pulse-load penalty
  • Coin cell self-discharge: ~1%/yr (negligible); AAA: ~2-3%/yr

Design Targets

  • 6-month target = acceptable battery life for e-ink v1
  • Module thickness: ≤10 mm total (display + battery + housing)
  • Baseline usage: 2 content updates per week
  • BLE-only architecture — Thread dropped (confirmed Akash + Aditya)

Fixed Specs & Developer Notes

Typography (Fixed)

  • Font family: IBM Plex Sans
  • Brand: 500 weight, uppercase
  • Flavour Name: 700 weight (bold)
  • Field Labels: 600 weight, uppercase
  • Field Values: 700 weight (bold)
  • Position labels: 600 weight, 75% of label size
  • Baseline offset from center: fontSize × 0.35

Layout (Fixed)

  • Padding: 15 px on all sides
  • Background: #FFFFFF, Text: #000000, Pos labels: #666666
  • Dividers: 2 px black between cards
  • Content clipped to card bounds (no overflow)
  • 1 coffee: full canvas, label FRONT
  • 2 coffees: L/R split, labels LEFT / RIGHT
  • 3–4: 2×2 grid, labels FRONT L/R, BACK L/R

Dot Rendering

  • Always 10 circles for intensity/bitterness
  • Filled ●: black fill + 1.5 px stroke
  • Empty ○: 1.5 px stroke only
  • “(N)” number: label font size, centered to dot row
  • Dot center Y = cursor (drawn top-aligned at cursor)

DECAF Badge

  • Shown right-aligned on BEST SERVED AS row
  • Only when coffee.decaf = true AND decaf field selected
  • Badge padding: 4 px horizontal, 2 px vertical
  • Border: 1.5 px black, square corners (no radius)
  • Text: same font/size as field labels (600 weight)

Spacing Rules (Derived)

  • All vertical spacing is center-to-center
  • Cursor tracks center of current element
  • Brand → Flavour: tight — brandSize/2 + 1 + nameSize/2
  • Flavour → first label: SEC + (nameSize − valueSize) / 2 (compensates for larger font)
  • Label → Value: LV pitch
  • Value → next Label: SEC pitch
  • Intensity & Bitterness rendered side-by-side (contentW / 2 split)

Field Ordering

  • Draw order: Brand → Flavour → Notes → Best Served As → Intensity/Bitterness row → Your Review
  • Decaf: inline badge on BEST SERVED AS row (not its own block)
  • Intensity/Bitterness: always on same row (left/right split)
  • Your Review: always last, text wraps to 2 lines, 75 char cap

Resolutions (Landscape)

  • 3.68": 792 × 528 px
  • 3.97": 800 × 480 px (default)
  • 3.98": 768 × 552 px
  • Preview canvas renders at target resolution (1×)
  • E-Ink Image preview uses 2× DPR for sharper onscreen display, then downscales for export

Bitmap Array Format (for firmware)

  • 1 bit per pixel, MSB first, row-major
  • White = 1, Black = 0
  • Luminance threshold: 0.299R + 0.587G + 0.114B ≥ 128
  • Total bytes = ceil(width × height / 8)
  • 180° rotated to match e-ink controller orientation (pixels read from bottom-right to top-left)
  • C header exports include WIDTH, HEIGHT, BYTES defines
